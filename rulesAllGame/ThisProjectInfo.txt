# ข้อมูลโปรเจค (Project Info) สำหรับ AI

โปรเจคนี้คือระบบ Backend สำหรับเกมไพ่แบบ Multiplayer แบบ Real-time ที่ถูกเขียนด้วย **Spring Boot (Java 17)** และใช้งาน **WebSocket (STOMP)** ในการสื่อสารระหว่าง Client และ Server 

ในโปรเจคนี้มีการรองรับการเล่นเกมหลักๆ 2 เกม ได้แก่ **Magic Pok-Pok** และ **Chaotic Auction** (รวมศูนย์อยู่ใน Game Hub)

---

## 1. Tech Stack และเครื่องมือที่ใช้
- **Framework:** Spring Boot 3.x (อ้างอิงจาก parent 4.0.5 ใน pom.xml)
- **Language:** Java 17
- **Communication:** WebSocket (`spring-boot-starter-websocket`) สำหรับส่งข้อมูล GameState แบบ Real-time และ RESTful API (`spring-boot-starter-web`)
- **Build Tool:** Maven (`pom.xml`)

---

## 2. โครงสร้างของโปรเจค (Project Structure)
อยู่ใน package หลัก `com.example.demo1` แบ่งเป็น:
- `model/`: คลาสที่ใช้เก็บสถานะและข้อมูลของเกม เช่น `Card`, `Player`, `GameState`, `GameMessage`, `ActionCommand`
- `controller/`: 
  - `GameWebSocketController`: รับ-ส่งข้อมูล WebSocket จาก Client (`@MessageMapping("/game.action")`)
  - `GameRestController`: API สำหรับการจัดการทั่วๆ ไป
- `service/`: Core Logic ของเกมทั้งหมด
  - `TurnEngineService`: จัดการระบบ Turn, การใช้ Skill ของการ์ด, กฎการลด Action, และ Effect ของการ์ดแต่ละใบ
  - `GameService`: จัดการ State ของห้องเกม การสร้างห้องและแจกไพ่
  - `EventProcessorService`: จัดการ Event ที่เกิดขึ้น เช่น การคำนวณคะแนน หรือเหตุการณ์ "A-Phet" เมื่อจบรอบ
  - `DeckFactory`: ใช้สร้างสำรับไพ่
- `auction/`: โฟลเดอร์แยกสำหรับระบบเกม "Chaotic Auction" (มี model/controller/service แยกของตัวเอง)
- `chat/`: โฟลเดอร์แยกสำหรับระบบแชท

## 3. รายละเอียดเกม Magic Pok-Pok
เป็นเกมไพ่แนววางแผนและต่อสู้ โดยมีระบบสำคัญๆ ดังนี้:

### โครงสร้างข้อมูล (GameState & Player)
- **Market (ตลาด):** การ์ดตรงกลางที่ผู้เล่นสามารถใช้ Action ดึงเข้า Board ตัวเองได้
- **Board:** พื้นที่วางไพ่ของผู้เล่น
- **Graveyard:** สุสานสำหรับไพ่ที่ถูกทำลายหรือใช้ไปแล้ว
- **Deck:** กองไพ่หลัก
- ผู้เล่นแต่ละคนมี **Action Point (Actions Left)** ปกติเริ่มที่ 2 แต้มต่อเทิร์น (อาจเพิ่ม/ลดจาก Effect ได้)

### ระบบ Skills ของไพ่ (ใน `TurnEngineService.java`)
ไพ่ถูกแบ่งเป็น 3 ประเภทคือ **Energy (A), Character (J, Q, K)** และ **Number (2-10)**
- **A (Energy):** แตะ (Tap) เพื่อใช้จ่ายเป็นคอสต์ต่างๆ
- **J (Destroyer):** ทำลายไพ่เบอร์ 2-10 ของศัตรู หรือพลีชีพ (Self-destruct) เพื่อทำลาย J, Q, K ของศัตรู
- **Q (Thief):** ขโมยไพ่จากบอร์ดศัตรู
- **K (Finisher):** เงื่อนไขชนะทันที (Sudden Death) ถ้ามี Action เหลือ 2 และมี A ที่ถูก Tap อยู่
- **2 (Curse):** ขโมยไพ่ Number ที่คะแนนสูงที่สุดของเป้าหมายและปิดการใช้งานไพ่ใบนั้น
- **3 (Silence):** ทำให้ศัตรูใช้ Skill ของ Character ไม่ได้
- **4 (Blind Draw):** จั่วไพ่ใบบนสุดจาก Deck
- **5 (Refresh):** ล้างตลาดและจั่วไพ่ใบใหม่มาวาง 5 ใบ
- **6 (Odd Dig):** ขุดไพ่เลขคี่ (3, 5, 7, 9) จาก Graveyard
- **7 (Slow):** ลด Action ของเป้าหมายในเทิร์นหน้าเหลือ 1
- **8 (Haste):** เพิ่ม Action ให้ตัวเอง +2 ในเทิร์นนี้
- **9 (Revive):** ขุดไพ่ Character (J, Q, K) ที่ตัวเองยังไม่มีจาก Graveyard
- **10 (Even Dig):** ขุดไพ่เลขคู่ (2, 4, 8) จาก Graveyard

### กฎกติกาพิเศษที่เขียนไว้แล้ว
- **Duplicate Character Rule:** ผู้เล่นไม่สามารถมีตัวละคร (J, Q, K) ตัวเดียวกันซ้ำบนบอร์ดได้ หากได้มาซ้ำ เกมจะบังคับให้เปิด Modal ทิ้งไพ่ (DISCARD_DUPLICATE) ก่อนทำแอคชั่นอื่น
- **A-Phet Sequence:** เมื่อวนครบ 1 รอบ (ทุกคนเล่นจบ) จะเกิด Event A-Phet ที่จะประกาศเหตุการณ์แบบสุ่มผ่านระบบ EventProcessor
- **Global Silence:** เอฟเฟกต์ใบ้หมู่ทั้งกระดาน

## 4. รายละเอียดเกม Chaotic Auction (ประมูลชุลมุน)
เป็นเกมประมูลไพ่ (2-6 คน) เล่นทั้งหมด 4 รอบใหญ่ โดยมีกติกาและกลไกหลักดังนี้:

### โครงสร้างแต่ละรอบย่อย (Round Flow)
1. **Master Card (ใบสั่งการ):** หงายไพ่ 1 ใบเพื่อกำหนด "เลขนำโชค" (+10 แต้ม) และ "ดอกนำโชค" (+20 แต้มสำหรับคนที่มีดอกนี้เยอะสุดตอนจบ) 
   - หากใบสั่งการเป็น 10, J, Q, K, A สีแดง ตลาดจะคว่ำหน้า (ประมูลปริศนา) 
   - หากใบสั่งการเป็น 10, J, Q, K, A สีดำ ตลาดจะเปิดเท่าจำนวนคน (ทุกคนได้ไพ่ชัวร์ แต่จะไม่มีโบนัส Underdog x2)
2. **การประมูล (Bidding):** ทุกคนลงไพ่คว่ำ 1 ใบและเปิดพร้อมกัน เรียงลำดับความใหญ่ K > Q > ... > 2 > A (ถ้าเลขเท่ากันดูดอก ♠ > ♥ > ♦ > ♣) **[อัปเดตล่าสุด: ไพ่ที่ใช้ประมูลชนะจะถูกทิ้งลงสุสาน ไม่นับเป็นคะแนน]**
3. **การสรุปผล (Resolution):**
   - **อันดับ 1:** หยิบไพ่ในตลาดก่อน 1 ใบ และได้ใช้ "พลังขุนพล"
   - **อันดับกลาง:** หยิบไพ่ที่เหลือตามลำดับ
   - **ที่โหล่ (Underdog):** ไม่ได้หยิบไพ่ในตลาด แต่เอาไพ่ที่ตัวเองใช้ประมูลไปเข้ากองคะแนน "คูณ 2" (ยกเว้นรอบสีดำ)

### พลังขุนพล (เฉพาะผู้ชนะอันดับ 1)
- **J (Jack):** ส่องกองจั่ว 2 ใบ เลือก 1 ใบสลับลงตลาด
- **Q (Queen):** จั่วไพ่ 1 ใบขึ้นมือ
- **K (King):** จั่วไพ่ 1 ใบลงกองคะแนนตัวเองฟรีๆ
- **A (Ace):** ถ้าเป็นที่โหล่ แต้มประมูลจาก 1 จะกลายเป็น 10 และได้ผล x2 (รวม 20 แต้มทันที)

### ระบบเชื่อมต่อรอบ (Legacy & Status)
เมื่อจบ 1 รอบใหญ่ (ไพ่หมดมือ):
- **Legacy Card:** เลือกไพ่ 1 ใบจากกองคะแนนขึ้นมือไว้ใช้รอบหน้า
- **เจ้าสัว (ที่ 1 จากรอบก่อน):** รอบหน้าจั่วไพ่เริ่มต้นน้อยลง 1 ใบ
- **ยาจก (ที่โหล่ จากรอบก่อน):** รอบหน้าจั่วไพ่เริ่มต้นเพิ่มขึ้น 1 ใบ
- ข้อมูลระบบนี้ถูกแยกเขียนไว้ใน package `auction`

## 5. เป้าหมายในการนำข้อมูลไปใช้ต่อ
เมื่อ AI รับไฟล์นี้ไป ควรอ่านเพื่อทำความเข้าใจ:
1. **Architecture** ของ WebSockets ที่รับคำสั่ง `ActionCommand` (เช่น TAKE_MARKET, USE_SKILL, END_TURN) เข้ามาประมวลผล
2. **Game State Management** ที่ต้อง Sync State กลับไปยังผู้เล่นทุกคนในห้อง (Topic: `/topic/game/{gameId}`)
3. **Logic Flow** ใน `TurnEngineService` ที่เป็นหัวใจของการออกท่าทาง (Skill) ของไพ่แต่ละใบ

หากต้องการปรับแต่งระบบต่อ ให้ยึดโครงสร้างและ Flow เดิมนี้เป็นหลักในการเพิ่มฟีเจอร์ใหม่ๆ
